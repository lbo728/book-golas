# 🚀 북골라스(LitGoal) 푸시 알림(Smart Nudge) 전략

> **작성일:** 2025년 12월 24일
> **작성자:** Senior PO (AI Assistant)
> **목표:** 단순 알림을 넘어 사용자의 독서 행동(Behavior)을 설계하는 넛지 아키텍처 수립

---

## 1. 핵심 철학 (Core Philosophy)

우리의 경쟁자는 다른 독서 앱이 아니라 **유튜브, 인스타그램, 넷플릭스**입니다. 사용자가 스마트폰을 켰을 때, 도파민을 자극하는 소셜 미디어 대신 북골라스를 선택하게 하려면 **"부담은 적게, 보상은 확실하게"** 느껴지는 메시지를 던져야 합니다.

### 3대 원칙
1.  **Personalized (개인화):** 모든 유저에게 같은 시간에 같은 메시지를 보내지 않는다.
2.  **Actionable (행동 유도):** 단순히 "읽으세요"가 아니라 "무엇을, 얼마나" 읽을지 제안한다.
3.  **Timely (적시성):** 사용자가 읽을 수 있는 상황(Context)을 공략한다.

## 2. 세그먼트별 푸시 시나리오 (Strategic Scenarios)

### A. 루틴 형성형 (The Habit Builder)
*사용자의 하루 일과 틈새를 공략하여 독서 습관을 형성합니다.*

| 타이밍 | 타겟 유저 | 메시지 전략 (Copywriting) | 데이터 로직 |
| :--- | :--- | :--- | :--- |
| **출근길/등교길**<br>(08:00~09:00) | 대중교통 이용 직장인/학생 | **"지옥철에서 벗어나는 법 📖"**<br>이어폰 꽂고 딱 10페이지만 읽어볼까요? | 오전 시간대 접속 이력 있는 유저 |
| **점심시간**<br>(12:30) | 직장인 | **"커피 한 잔과 책 한 줌 ☕️"**<br>남은 점심시간 15분, [책제목]과 함께 리프레시하세요. | 직장인 추정 그룹 (평일 낮 활동) |
| **취침 전**<br>(22:00~23:00) | 자기 전 스마트폰 보는 유저 | **"오늘 하루의 마무리는"**<br>스마트폰 불빛 대신 [책제목]의 문장으로 채워보세요. 🌙 | 심야 시간대 접속 이력 있는 유저 |

### B. 목표 관리형 (The Pacer)
*북골라스의 핵심 기능인 '목표일(Target Date)'과 '남은 분량'을 활용한 초개인화 알림입니다.*

| 상황 | 메시지 전략 (Copywriting) | 데이터 로직 |
| :--- | :--- | :--- |
| **페이스 저조**<br>(Goal Behind) | **"🚨 목표 달성에 빨간불이 켜졌어요"**<br>이번 주말까지 하루 [N]페이지만 더 읽으면 따라잡을 수 있어요! | `(남은 페이지 / 남은 일수)` > `평균 독서 속도` |
| **완독 임박**<br>(Almost There) | **"결승선이 코앞이에요! 🏁"**<br>딱 [N]페이지만 더 읽으면 [책제목] 완독! 오늘 끝내버릴까요? | 잔여 분량 < 전체의 10% |
| **작심삼일 방지**<br>(3-Day Drop) | **"책갈피에 먼지가 쌓이고 있어요 😢"**<br>[책제목]의 주인공이 3일째 기다리고 있습니다. | 마지막 기록 후 72시간 경과 |

### C. 성취감 고취형 (The Cheerleader)
*사용자의 행동에 대해 즉각적인 보상(칭찬)을 제공하여 도파민을 생성합니다.*

| 상황 | 메시지 전략 (Copywriting) | 데이터 로직 |
| :--- | :--- | :--- |
| **연속 기록**<br>(Streak) | **"🔥 7일 연속 독서 달성!"**<br>상위 5%의 독서 습관을 가지고 계시네요. 이 기세 그대로! | Streak 3, 7, 14, 30일 도달 시 |
| **페이지 돌파** | **"벌써 100페이지나 읽으셨네요!"**<br>천리길도 한 걸음부터, 벌써 산 중턱에 도착했어요 ⛰️ | 누적 페이지 100단위 돌파 시 |
| **주말 결산**<br>(Sunday Night) | **"이번 주, 마음의 양식을 얼마나 쌓았을까요?"**<br>총 [N]페이지를 읽으셨어요. 당신의 한 주를 확인해보세요 📊 | 매주 일요일 저녁 20:00 |

## 3. 기술적 실행 계획 (Implementation Roadmap)

### Phase 1: 정적 룰 베이스 (Static Rules) - *현재 단계*
*   **Target:** 모든 유저 또는 단순 조건 유저.
*   **Action:**
    *   매일 20:00 고정 알림 (기존 `daily-nudge.yml` 활용).
    *   "3일 미접속 유저" 타겟팅 추가 (Supabase Edge Function에서 쿼리로 필터링).

### Phase 2: 동적 템플릿 (Dynamic Templates)
*   **Target:** 개별 유저의 데이터(책 제목, 남은 페이지) 반영.
*   **Action:**
    *   FCM Payload에 `data` 필드를 활용하여 클라이언트에서 메시지 구성.
    *   *예: "철수님, [해리포터] 완독까지 50페이지 남았어요!"*

### Phase 3: AI/ML 최적화 (Smart Nudge)
*   **Target:** 개인별 최적 시간(Optimal Send Time).
*   **Action:**
    *   유저가 앱을 여는 시간대를 Supabase DB에 로깅.
    *   해당 시간대 ±30분에 맞춰 개별 큐잉(Queueing) 발송.

## 4. 성공 지표 (Metrics)

*   **CTR (Click-Through Rate):** 푸시 알림 클릭률 (목표: 5% 이상)
*   **Session Conversion:** 푸시 진입 후 실제 '독서 기록(Page Update)' 발생 비율.
*   **Retention D+30:** 푸시 수신 동의 유저의 30일 후 잔존율.

## 5. 심화 전략 (Next Level Strategy - Deep Dive)

기능적인 알림을 넘어, 사용자의 심리적 장벽을 낮추고 이탈을 방지하는 고도화된 전략입니다.

### 1. "시간 화폐" 전략 (Time Currency Nudge)
사용자는 '페이지 수'보다 '시간'에 더 민감하게 반응합니다. 막연한 분량을 만만한 시간으로 치환합니다.

*   **Insight:** "50페이지 남음" (많아 보임) vs "15분이면 완독" (할 만해 보임).
*   **Logic:** `(남은 페이지 / 유저의 평균 분당 페이지 수)`를 계산하여 시간으로 환산.
*   **Message:** "남은 50페이지, **딱 18분**이면 다 읽을 수 있어요. 자기 전 18분만 투자해보세요."

### 2. "중단점 회상" 전략 (Recap Nudge)
책을 펴는 것이 두려운 이유 중 하나는 '앞 내용이 기억나지 않아서'입니다.

*   **Insight:** 독서의 맥락(Context)을 끊어지지 않게 이어주어야 합니다.
*   **Logic:** 사용자가 마지막으로 남긴 메모나 하이라이트가 있다면 이를 인용.
*   **Message:** "지난번에 '이 문장이 인상 깊다'고 메모하셨죠? 📝 그 다음 이야기가 기다리고 있어요."

### 3. "침묵의 미학" 전략 (Anti-Nudge Protocol)
가장 중요합니다. 맹목적인 알림은 스팸입니다. 반응이 없으면 전략을 바꿔야 합니다.

*   **Insight:** 알림을 계속 무시하는 건 '바빠서'가 아니라 '그 책이 재미없어서'일 확률이 높습니다. 죄책감을 덜어줘야 합니다.
*   **Logic:** 알림 반응률(CTR)이 0%인 상태가 3회 이상 지속될 경우.
*   **Message:** "요즘 [책제목] 읽기가 힘드신가요? 📚 잠시 보관함으로 옮기고, 가벼운 에세이로 기분 전환하는 건 어때요?"
